/server/normal 正常节点
    - /launch 开启 √
    - /restore_and_launch 恢复增量备份并启动(如果没有,则就是正常启动) √
    - /launch_and_discard_backup 丢弃残留的增量备份(如果没有,则就是正常启动) √

/server/maintenance 维护节点（部署后不会监控Minecraft服务器）
    - /launch 开启 √ 
    - /stop 关闭（软），会等待Minecraft服务器进程正常结束后打包服务器 √
    - /kill 关闭（硬） √
    - /get_key 获得本次服务器登入私匙 √
        - 返回数据在data对象中的privateKey字段中
    - /revive 清除当前错误，继续运行(仅用于少数偶发性错误)，仅在发生错误时可用 √
    - /wipe_butt 无法revive的情况下，直接退还资源(擦屁股)（调用的cleanDeploy）√
        【警告】本操作无法撤销，只能在无法revive的情况下调用。
    - /restore_and_launch 恢复增量备份并启动(如果没有,则就是正常启动) √
    - /launch_and_discard_backup 丢弃残留的增量备份(如果没有,则就是正常启动) √

/server/command 命令节点
    - /send 发送命令 √
        - 方式：POST，表单字段包含 command - 命令内容

/server/mc_logs 日志获取节点(特殊)
    - /get 获得日志 √

/server/query 查询节点
    - /mc 查询Minecraft服务器状态 √
    - /backend 查询后端服务状态 √

/backend/statistics 统计节点
    - /costs 查询服务器费用
    - /launches 启动情况

/backend/token 临时api请求token生成的节点
    - /generate 生成游客访问token
        - 方式：POST，表单字段包含 validity - token有效时间(毫秒)，可以不传入

> 返回数据中code为0代表异步处理，为1则代表成功，为-1代表失败

提示信息：
Invalid Request - 无效的请求
Invalid Path - 无效的请求路径
Lack of Valid Action - 缺少有效的请求参数?act
Non-existent Node - 访问了不存在的操作节点，比如/server/iehfihfi
Permission Denied - 访问了错误的路径或者token权限不够

status_codes.json配置的inform仅供参考，具体是否通知取决于实现。
